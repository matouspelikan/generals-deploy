## Exercise 11.1

The cluster hypothesis says that similar documents tend to be relevant to the same information needs. With the proposed similarity definition, two documents are similar if they share at least two proper names.

Counterexample information need:

```text
Find documents about scandals involving Bill Clinton and Nicolas Sarkozy.
```

Document 1:

```text
Bill Clinton and Nicolas Sarkozy met at an international summit to discuss climate policy.
```

Document 2:

```text
Bill Clinton and Nicolas Sarkozy were both mentioned in a retrospective article about political scandals.
```

The documents share the proper names `Clinton` and `Sarkozy`, so they are similar by the proposed definition. But only Document 2 is relevant to the information need. The shared names alone do not capture the user's intended relation or topic.

## Exercise 11.2

Consider one-dimensional documents on a line:

```text
Cluster A: 0, 100
Cluster B: 49, 51
Query: 1
```

If we retrieve by direct nearest neighbor search, the nearest document to the query is:

```text
0
```

which is in Cluster A.

But if cluster-based retrieval represents each cluster by its centroid:

```text
centroid(A) = (0 + 100) / 2 = 50
centroid(B) = (49 + 51) / 2 = 50
```

Cluster A and Cluster B look equally close by centroid, even though Cluster A contains the true nearest neighbor. With a small perturbation:

```text
Cluster A: 0, 102     centroid = 51
Cluster B: 48, 50     centroid = 49
Query: 1
```

Cluster B's centroid is closer to the query, but the nearest individual document is still `0` in Cluster A. This shows the inexactness of cluster-based retrieval: a cluster summary can hide a very close document.

## Exercise 11.3

The original slide has 17 points and 3 clusters. From the slide's counts:

```text
Cluster 1: x = 5, o = 1
Cluster 2: o = 4, diamond = 2
Cluster 3: x = 2, diamond = 3
```

The original purity is:

```text
(5 + 4 + 3) / 17 = 12/17 ≈ 0.71
```

### Difficulty after doubling

Replacing every point by two identical copies in the same class does not add new structure. Conceptually, it is equally difficult to cluster: the data set contains the same distinct point locations and labels, just duplicated.

### Purity after doubling

After doubling:

```text
Cluster 1: x = 10, o = 2
Cluster 2: o = 8, diamond = 4
Cluster 3: x = 4, diamond = 6
```

The majority counts are:

```text
10, 8, 6
```

So:

```text
purity = (10 + 8 + 6) / 34
       = 24 / 34
       = 12 / 17
       ≈ 0.71
```

Purity stays the same.

### Rand index after doubling

For the doubled data, the true positives are the same-class pairs inside clusters:

```text
TP = C(10,2) + C(2,2) + C(8,2) + C(4,2) + C(4,2) + C(6,2)
   = 45 + 1 + 28 + 6 + 6 + 15
   = 95
```

Pairs in the same cluster:

```text
TP + FP = C(12,2) + C(12,2) + C(10,2)
        = 66 + 66 + 45
        = 177
```

So:

```text
FP = 177 - 95 = 82
```

Class sizes after doubling are:

```text
x = 14, o = 10, diamond = 10
```

Same-class pairs:

```text
TP + FN = C(14,2) + C(10,2) + C(10,2)
        = 91 + 45 + 45
        = 181
```

So:

```text
FN = 181 - 95 = 86
```

Total pairs:

```text
C(34,2) = 561
```

Therefore:

```text
TN = 561 - TP - FP - FN
   = 561 - 95 - 82 - 86
   = 298
```

Rand index:

```text
RI = (TP + TN) / (TP + FP + FN + TN)
   = (95 + 298) / 561
   = 393 / 561
   ≈ 0.701
```

The original slide gives:

```text
RI_original = (20 + 72) / 136 ≈ 0.676
```

So after doubling:

```text
purity stays the same
RI increases
```

Since the doubled problem is not really a different clustering problem, an evaluation measure that stays the same is better suited for comparing these two clusterings. In this specific comparison, purity is preferable to RI. More generally, though, purity has its own problem: it can be artificially increased by using more clusters.

## Exercise 11.4

Documents that use different terms for `car`, such as `car` and `automobile`, can still end up in the same K-means cluster because clustering uses all vector dimensions, not only the exact word `car`.

For example, documents about cars and automobiles may share many other terms:

```text
dealer, engine, road, wheel, fuel, repair, Mercedes, vehicle
```

Even if one document says `car` and another says `automobile`, their full document vectors can be close because of these shared contextual terms. K-means groups documents by distance to centroids, so shared context can pull them into the same cluster.

## Exercise 11.5

The two termination conditions are:

1. assignments do not change
2. centroids do not change

If assignments do not change, then centroids do not change, because each centroid is computed as the average of exactly the same assigned points.

The reverse is not logically guaranteed in all possible data sets. Different assignments can produce the same centroid, especially when there are duplicate points or symmetric configurations. For example, two identical documents could swap between two clusters without changing either centroid.

In normal K-means with deterministic tie-breaking, unchanged centroids imply that the next reassignment step will also be unchanged, because the same points are compared with the same centroids. So as practical stopping tests they are closely related, but as logical statements about arbitrary assignments, only:

```text
unchanged assignments => unchanged centroids
```

is always true.

## Exercise 11.6

For single-link and complete-link clustering, cluster similarity is the maximum or minimum of original document-document similarities. Therefore there can be at most about `N^2` distinct similarity values, because every cluster similarity is one of the original pairwise document similarities.

For centroid clustering and GAAC, cluster similarity is not restricted to one original document-document similarity. It is computed from averages or centroids of clusters. Different subsets of documents can produce new similarity values.

The number of possible nonempty disjoint cluster pairs is exponential. Each document can be:

```text
in the first cluster,
in the second cluster,
or in neither cluster
```

so there are on the order of:

```text
3^N
```

possible ordered disjoint cluster pairs, and therefore exponentially many possible cluster similarities. More precisely, ignoring order and excluding empty clusters, the count is:

```text
(3^N - 2 * 2^N + 1) / 2
```

So GAAC and centroid clustering can have exponentially many distinct cluster similarities, not merely `N^2`.

## Exercise 11.7

For 2-means on documents from two different languages, I would expect the two clusters to mostly correspond to the two languages.

Reason: documents in the same language share many function words and vocabulary items, while documents in different languages often have almost disjoint vocabularies. In vector space, within-language similarities will usually be much higher than cross-language similarities.

For HAC, I would also often expect the hierarchy to separate by language near the top, especially with complete-link or GAAC. But the result depends more on the linkage method:

- Single-link can be affected by chaining through shared names, numbers, URLs, or borrowed words.
- Complete-link tends to avoid loose cross-language chains.
- GAAC is often a good choice for vector representations and should usually preserve the language split.

So the broad expectation is the same, but HAC's exact behavior depends on the cluster similarity criterion.
