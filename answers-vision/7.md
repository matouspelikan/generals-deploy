# Lecture 7 Answers

## Exercise 7.1

Rocchio's SMART relevance-feedback formula is:

```text
q_m = alpha * q_0 + beta * mu(D_r) - gamma * mu(D_nr)
```

A "Find pages like this one" search has no real original query and only one positive example document. It corresponds to:

```text
alpha = 0
beta  = 1
gamma = 0
```

That is, the modified query vector is just the vector of the selected page, or equivalently the centroid of the one relevant document:

```text
q_m = mu(D_r)
```

If there are several pages selected as examples, then `q_m` would be their centroid.

## Exercise 7.2

Three reasons relevance feedback has been little used in web search:

1. Users are reluctant to give explicit relevance judgments. Marking documents relevant/nonrelevant is extra work compared with reformulating the query.
2. Relevance feedback can create long expanded queries, and long queries are expensive to process at web scale.
3. The results can be hard for users to understand. After feedback, a page may be retrieved because of terms added by the algorithm rather than terms the user typed.

Another practical reason is that the same user effort may be better spent revising and resubmitting the query. The slides note that there is no clear evidence that explicit relevance feedback is the best use of the user's time in web search.

## Exercise 7.3

Positive feedback is usually more useful because relevant documents tend to say what the user is actually looking for. Their terms can be added or upweighted to move the query toward the relevant-document centroid.

Negative feedback is less reliable because there are many ways for a document to be nonrelevant. A nonrelevant document may be off-topic, too broad, about a different sense of a word, in the wrong genre, or simply not useful for some other reason. Moving away from many different nonrelevant documents can push the query in an unstable or unhelpful direction.

Using only one nonrelevant document can sometimes be better than using several because it may remove one clear mistaken interpretation of the query without overgeneralizing. Several nonrelevant documents may be heterogeneous, so their centroid may not represent a coherent "bad direction" to move away from.
