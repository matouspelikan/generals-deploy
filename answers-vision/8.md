## Exercise 8.1

With no document relevance information available, BIM is usually estimated with:

```text
p_t = P(x_t = 1 | relevant, q) = 0.5
u_t = P(x_t = 1 | nonrelevant, q) approx df_t / N
```

This makes the BIM score essentially:

```text
RSV_d = sum over query terms present in d of log(N / df_t)
```

So BIM becomes operationally similar to idf-weighted Boolean matching, but it differs from standard vector-space tf-idf in several ways:

| aspect | vector-space tf-idf | BIM with no relevance info |
| --- | --- | --- |
| Representation | Usually weighted term vectors | Binary incidence vectors |
| Term occurrence | Uses term frequency, often `1 + log(tf)` | Only records whether a query term is present |
| Scoring interpretation | Geometric similarity, usually cosine | Probability-motivated retrieval status value |
| Length normalization | Commonly uses cosine or pivoted normalization | Basic BIM has no document length normalization |
| Term weights | tf multiplied by idf | idf-like probabilistic weight derived from odds |
| Query/document model | Query and document are vectors in a space | Relevance is modeled probabilistically |

The main practical similarity is that both can be implemented using an inverted index and both give higher weight to rare query terms. The main difference is that standard tf-idf uses within-document term frequency and vector normalization, while basic BIM is binary and probabilistic.

## Exercise 8.2

Let:

```text
n = |R|
s = number of relevant documents where X_t = 1
p_t = P(X_t = 1 | R = 1, q)
```

The observations for the relevant set are Bernoulli trials. We observe `s` successes, where the term appears, and `n - s` failures, where it does not.

The likelihood is:

```text
L(p_t) = p_t^s * (1 - p_t)^(n - s)
```

It is easier to maximize the log-likelihood:

```text
ell(p_t) = s log p_t + (n - s) log(1 - p_t)
```

Differentiate:

```text
d ell / d p_t = s / p_t - (n - s) / (1 - p_t)
```

Set the derivative to zero:

```text
s / p_t = (n - s) / (1 - p_t)
```

Cross-multiply:

```text
s(1 - p_t) = (n - s)p_t
s - s p_t = n p_t - s p_t
s = n p_t
p_t = s / n
```

Since `n = |R|`, the maximum likelihood estimate is:

```text
p_t = s / |R|
```

So the MLE is the observed fraction of relevant documents that contain term `t`.
