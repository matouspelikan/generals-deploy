# Lecture 5 Answers

## Key Concepts

**Boolean retrieval vs. ranked retrieval:** Boolean retrieval returns an unordered set of documents that exactly match a logical query — documents are either in or out. Ranked retrieval scores every document and returns a sorted list, allowing partial matches and enabling the system to prioritize more relevant results.

**Document-query scoring:** assigning a numeric relevance score to each document with respect to a query, typically by combining term weights. The score determines the document's rank in the result list.

**Bag-of-Words (BoW) model:** represents a document (or query) as an unordered multiset of its terms, ignoring word order and grammar. Simple and effective for many IR tasks.

**Term weighting:** assigning a numeric weight to each term in a document or query to reflect its importance. Higher weight means the term is more discriminative or more frequent.

**Term frequency (tf):** the number of times a term appears in a document. Often log-scaled: tf = 1 + log(count) to dampen the effect of very high raw counts.

**Document frequency (df):** the number of documents in the collection that contain a given term. High df means the term is common and less discriminative.

**TF-IDF:** a term-weighting scheme combining tf (how often a term appears in a document) and idf (inverse document frequency = log(N/df), how rare the term is across the collection). tf-idf is high when a term is frequent in a document but rare across the collection.

**Vector space model:** represents both documents and queries as vectors in a high-dimensional term space. Relevance is measured by the similarity between the query vector and each document vector (typically cosine similarity).

**Similarity in vector space:** usually measured by cosine similarity — the cosine of the angle between two vectors. Ranges from 0 (no overlap) to 1 (identical direction), and is not affected by document length.

**Length normalization:** dividing a document vector by its Euclidean length so that all document vectors become unit vectors. Prevents long documents from scoring higher simply because they contain more terms.

**Pivoted length normalization:** a variant that normalizes less aggressively, using a pivot (the average document length) to avoid over-penalizing genuinely long documents that are relevant.

## Exercise 5.1

> What is the idf of a term that occurs in every document? Compare this with the use of stop word lists.

The idf of a term that occurs in every document is:

```text
idf_t = log10(N / df_t) = log10(N / N) = log10(1) = 0
```

So a term that appears in every document receives weight 0. This is similar in spirit to stop word removal: very frequent terms carry little or no discriminative information. The difference is that idf handles this continuously by giving frequent terms low weight, while a stop word list removes selected terms completely.

## Exercise 5.3

> Can the tf-idf weight of a term in a document exceed 1?

Yes. With the formula above, tf-idf is not restricted to the interval `[0, 1]`.

For example, in Exercise 5.2:

```text

```

The value can exceed 1 because both the logarithmic tf weight and the idf weight can be greater than 1.

## Exercise 5.4

> How does the base of the logarithm in idf affect the calculation of the tf-idf score for a given query and a given document? How does the base of the logarithm affect the relative scores of two documents on a given query?

Changing the logarithm base changes every idf value by the same positive constant. For example:

```text
log_b(x) = log_a(x) / log_a(b)
```

So switching from base `a` to base `b` rescales idf by `1 / log_a(b)`.

For a fixed query-document pair, this rescales the tf-idf score by a constant factor, assuming the same weighting formula is used consistently. If idf appears once in each term contribution, the score is multiplied by one positive constant; if idf appears in both query and document weights, it is multiplied by a positive constant squared.

The relative ranking of two documents for the same query is not affected, because multiplying all scores by the same positive constant preserves their order. With cosine normalization, even the vector directions are unchanged by a global positive scaling.

## Exercise 5.5

> If we were to stem jealous and jealousy to a common stem before setting up the vector space, detail how the definitions of tf and idf should be modified.

If `jealous` and `jealousy` are stemmed to a common stem, say `jealous`, then the vector space should be built over stems rather than original surface words.

The term frequency becomes the sum of occurrences of all forms mapped to that stem:

```text
tf_jealous,d = count(jealous in d) + count(jealousy in d)
```

More generally, if a stem `s` has variants `v1, ..., vk`, then:

```text
tf_s,d = sum_i tf_vi,d
```

The document frequency is the number of documents containing at least one of the variants:

```text
df_s = |{d : tf_s,d > 0}|
```

This is not necessarily the sum of the individual document frequencies, because a document may contain both `jealous` and `jealousy`. Then:

```text
idf_s = log10(N / df_s)
```

Queries must be stemmed the same way.

## Exercise 5.7

> Verify that the sum of the squares of the components of each of the document vectors in Exercise 5.6 is 1 (to within rounding error). Why is this the case?

Using the normalized vectors from Exercise 5.6:

```text
Doc1: 0.6696^2 + 0.5128^2 + 0.0000^2 + 0.5373^2 ≈ 1.0000
Doc2: 0.3699^2 + 0.7330^2 + 0.5709^2 + 0.0000^2 ≈ 1.0000
Doc3: 0.6022^2 + 0.0000^2 + 0.6117^2 + 0.5130^2 ≈ 1.0000
```

This is true because Euclidean normalization divides each vector by its own Euclidean length:

```text
||d||_2 = sqrt(sum_i d_i^2)
```

After division by `||d||_2`, the new vector has length 1, so the sum of squared components is 1.

## Exercise 5.9

> Show that if a query vector and document vectors are all normalized to unit vectors, then the rank ordering produced by Euclidean distance is identical to that produced by cosine similarities.

Assume query vector `q` and document vector `d` are both unit vectors:

```text
||q|| = ||d|| = 1
```

Then:

```text
||q - d||^2 = ||q||^2 + ||d||^2 - 2(q · d)
            = 1 + 1 - 2(q · d)
            = 2 - 2 cos(q, d)
```

So Euclidean distance is a decreasing function of cosine similarity:

```text
||q - d|| = sqrt(2 - 2 cos(q, d))
```

Minimizing Euclidean distance is therefore equivalent to maximizing cosine similarity. The two methods produce the same ranking.

## Exercise 5.10

> In turning a query into a unit vector we can assign equal weights to each of the query terms. What other principled approaches are plausible?

Equal weights for all query terms are simple, but other principled choices are possible:

- Use idf weights, so rare query terms receive more weight than common query terms.
- Use tf-idf query weights, especially if the query repeats a term.
- Use logarithmic tf weighting for query terms, as with documents.
- Use feedback-based weights, for example Rocchio relevance feedback or pseudo-relevance feedback.
- Use field or phrase information, giving extra weight to exact phrases or terms entered in important query fields.

The common principle is that query terms should be weighted by how informative they are and how strongly they express the user's information need.
