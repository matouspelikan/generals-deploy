## Key Concepts

**Vector space classification:** represents documents as vectors (same tf-idf vectors used in retrieval) and classifies them based on their position in vector space. Documents close to each other should belong to the same class.

**k-nearest neighbor (k-NN) classification:** classifies a test document by finding the k training documents closest to it (by cosine or Euclidean distance) and assigning the majority class among those neighbors. Non-parametric and requires no training phase but is slow at test time (O(|training set|) per query).

**k-NN complexity:** test time is O(|training set| × |vocabulary|) per document, which is expensive for large collections. Can be sped up with approximate nearest-neighbor structures.

**Probabilistic k-NN:** instead of majority vote, estimates P(class | document) from the fraction of the k neighbors belonging to each class, enabling soft/probabilistic class assignments.

**Linear classifiers:** classifiers that separate classes with a hyperplane: w·x + b = 0. Documents on one side are assigned to one class, documents on the other side to the other. Fast at test time, but can only separate linearly separable classes.

**Non-linear classifiers:** classifiers (e.g. k-NN, kernel SVMs, neural networks) that can learn curved decision boundaries, allowing them to classify non-linearly separable data.

**Multi-class classification — one-of:** each document is assigned exactly one class from a mutually exclusive set. Typically implemented with a softmax output or by reducing to multiple binary classifiers (one-vs-rest or one-vs-one).

**Multi-class classification — any-of:** a document can belong to any subset of classes simultaneously. Implemented as one independent binary classifier per class.

**Support Vector Machines (SVMs):** linear classifiers that find the hyperplane with the maximum margin (distance to the nearest training points from either class). This large-margin principle leads to good generalization.

**Large margin classification:** SVMs maximize the margin between the two classes. Support vectors are the training points lying exactly on the margin boundaries; all other points do not affect the decision boundary.

**Linear separability:** a dataset is linearly separable if a hyperplane exists that perfectly divides the two classes. SVMs can handle non-separable data via slack variables or kernel functions.

**Slack variables:** introduced in soft-margin SVMs to allow some training points to violate the margin or even be misclassified. The trade-off between margin size and violation penalty is controlled by a parameter C.

## Exercise 10.1

> Prove that the number of linear separators of two classes is either infinite or zero.

For a fixed finite training set, the number of linear separators is either zero or infinite.

If the two classes are not linearly separable, then there are no linear separators, so the number is zero.

If there is one linear separator, then there are infinitely many. A linear separator can be written as:

```text
w · x + b = 0
```

For strict separation, every training point has a positive margin from the separating hyperplane. Since the training set is finite, there is a smallest positive margin. We can therefore move the hyperplane slightly, or perturb `w` and `b` slightly, without crossing any training point. All sufficiently small perturbations still separate the two classes. There are infinitely many such perturbations.

Therefore the number of linear separators is either:

```text
0, if no separator exists
infinite, if at least one separator exists
```

## Exercise 10.2

> Which of the three vectors a = (0.5, 1.5), x = (2, 2), b = (4, 4), and c = (8, 6) is:
>
> 1. most similar to x according to dot product similarity
> 2. most similar to x according to cosine similarity
> 3. closest to x according to Euclidean distance?

The vectors are:

```text
a = (0.5, 1.5)
x = (2, 2)
b = (4, 4)
c = (8, 6)
```

### Dot product similarity

```text
a · x = 0.5*2 + 1.5*2 = 4
b · x = 4*2 + 4*2 = 16
c · x = 8*2 + 6*2 = 28
```

Most similar by dot product:

```text
c
```

Dot product favors long vectors, so `c` wins.

### Cosine similarity

Cosine similarity compares angle, not length.

```text
cos(a, x) = 4 / (sqrt(0.5^2 + 1.5^2) * sqrt(2^2 + 2^2))
          ≈ 0.894

cos(b, x) = 16 / (sqrt(4^2 + 4^2) * sqrt(2^2 + 2^2))
          = 1

cos(c, x) = 28 / (sqrt(8^2 + 6^2) * sqrt(2^2 + 2^2))
          ≈ 0.990
```

Most similar by cosine:

```text
b
```

`b` points in exactly the same direction as `x`.

### Euclidean distance

```text
dist(a, x) = sqrt((0.5 - 2)^2 + (1.5 - 2)^2)
           = sqrt(2.5)
           ≈ 1.581

dist(b, x) = sqrt((4 - 2)^2 + (4 - 2)^2)
           = sqrt(8)
           ≈ 2.828

dist(c, x) = sqrt((8 - 2)^2 + (6 - 2)^2)
           = sqrt(52)
           ≈ 7.211
```

Closest by Euclidean distance:

```text
a
```

## Exercise 10.3

> What is the minimum number of support vectors that there can be for a data set (which contains instances of each class)?

The minimum number of support vectors is `2`, assuming the data set contains instances of both classes.

Reason: a separating hyperplane must be constrained from both sides. At least one point from the positive class and at least one point from the negative class must lie on the margin; otherwise the margin could be increased by moving the separator.

So the minimum is:

```text
one support vector from each class = 2 support vectors
```
