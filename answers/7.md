## Key Concepts

**Presenting IR results:** how results are displayed to the user — typically as a ranked list with title, URL, and a short snippet. Good presentation helps users quickly identify relevant documents without opening each one.

**Summaries:** condensed representations of a document shown in the result list. Can be static (fixed extract from the document) or dynamic (generated to reflect the query).

**Snippets:** short dynamic summaries that show the query terms in context, typically highlighting the matched terms. Help users judge relevance at a glance.

**Relevance feedback:** a technique where the user explicitly marks some retrieved documents as relevant or nonrelevant, and the system uses these judgments to reformulate the query and retrieve better results.

**Rocchio algorithm:** the classic relevance feedback method that modifies the query vector by moving it toward the centroid of relevant documents and away from the centroid of nonrelevant documents: q_m = α·q₀ + β·μ(D_r) − γ·μ(D_nr).

**Positive vs. negative feedback:** positive feedback (marking relevant documents) is generally more useful than negative feedback (marking nonrelevant ones), because relevant documents share coherent topical features whereas nonrelevant documents can be off-topic for many different reasons.

**Pseudo-relevance feedback (blind feedback):** automatically treats the top-k retrieved documents as relevant and uses them to expand the query, without asking the user. Improves recall on average but can go wrong if the top results are not actually relevant.

**Query expansion:** augmenting the user's original query with additional related terms to improve recall. Terms can come from relevance feedback, a thesaurus, or co-occurrence statistics.

**Thesaurus-based query expansion:** adding synonyms or related terms from a manually constructed or automatically derived thesaurus (e.g. WordNet). Controlled and predictable but limited by thesaurus coverage.

**Co-occurrence-based query expansion:** finding terms that co-occur frequently with query terms in the collection and adding them to the query. Adapts to the specific collection but can introduce noise.

## Exercise 7.1

> In Rocchio's 1971 algorithm (SMART), what weight setting for α/β/γ does a "Find pages like this one" search correspond to?

Rocchio's SMART relevance-feedback formula is:

```text
q_m = alpha * q_0 + beta * mu(D_r) - gamma * mu(D_nr)
```

A "Find pages like this one" search has no real original query and only one positive example document. It corresponds to:

```text
alpha = 0
beta  = 1
gamma = 0
```

That is, the modified query vector is just the vector of the selected page, or equivalently the centroid of the one relevant document:

```text
q_m = mu(D_r)
```

If there are several pages selected as examples, then `q_m` would be their centroid.

## Exercise 7.2

> Give three reasons why relevance feedback has been little used in web search.

Three reasons relevance feedback has been little used in web search:

1. Users are reluctant to give explicit relevance judgments. Marking documents relevant/nonrelevant is extra work compared with reformulating the query.
2. Relevance feedback can create long expanded queries, and long queries are expensive to process at web scale.
3. The results can be hard for users to understand. After feedback, a page may be retrieved because of terms added by the algorithm rather than terms the user typed.

Another practical reason is that the same user effort may be better spent revising and resubmitting the query. The slides note that there is no clear evidence that explicit relevance feedback is the best use of the user's time in web search.

## Exercise 7.3

> Why is positive feedback likely to be more useful than negative feedback to an IR system? Why might only using one nonrelevant document be more effective than using several?

Positive feedback is usually more useful because relevant documents tend to say what the user is actually looking for. Their terms can be added or upweighted to move the query toward the relevant-document centroid.

Negative feedback is less reliable because there are many ways for a document to be nonrelevant. A nonrelevant document may be off-topic, too broad, about a different sense of a word, in the wrong genre, or simply not useful for some other reason. Moving away from many different nonrelevant documents can push the query in an unstable or unhelpful direction.

Using only one nonrelevant document can sometimes be better than using several because it may remove one clear mistaken interpretation of the query without overgeneralizing. Several nonrelevant documents may be heterogeneous, so their centroid may not represent a coherent "bad direction" to move away from.
