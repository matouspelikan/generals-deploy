## Key Concepts

**Importance of ranking in IR:** users typically examine only the top few results, so returning the most relevant documents first is more important than maximizing total recall. A good ranking function is central to user satisfaction.

**Computing top-k results:** instead of scoring all documents, efficient IR systems use techniques like early termination, skip pointers, and champion lists to compute only the top-k results without visiting every posting.

**Composite scores (PageRank):** combining query-dependent relevance scores (e.g. tf-idf cosine) with query-independent quality signals (e.g. PageRank, which measures a page's authority based on the link graph) to produce a final ranking score.

**Document-at-a-time (DAAT) processing:** computing the full score for one document before moving to the next. Allows early termination when a document's score cannot beat the current top-k threshold.

**Term-at-a-time (TAAT) processing:** accumulating scores for all documents one query term at a time. Requires keeping accumulators for all potentially relevant documents in memory.

**Tiered indexes:** organizing postings lists into tiers of decreasing quality (e.g. tier 1 contains only the most important postings). The system first searches tier 1, and only falls back to lower tiers if not enough results are found.

**Evaluation:** the systematic measurement of IR system quality by comparing retrieved results against known relevant documents using standardized metrics.

**Document relevance:** a document is relevant if it satisfies the information need behind the query. Relevance is typically assessed by human judges and recorded in a *qrels* file.

**Confusion matrix:** a 2×2 table showing the counts of true positives (relevant, retrieved), false positives (nonrelevant, retrieved), false negatives (relevant, not retrieved), and true negatives (nonrelevant, not retrieved).

**Precision:** the fraction of retrieved documents that are relevant: P = TP / (TP + FP). Measures how much noise is in the result set.

**Recall:** the fraction of relevant documents that are retrieved: R = TP / (TP + FN). Measures how much of the relevant set is found.

**F-measure:** the weighted harmonic mean of precision and recall. F₁ = 2PR/(P+R) gives equal weight to both; Fβ weights recall β² times more than precision.

**Precision-recall trade-off:** increasing recall (retrieving more documents) typically lowers precision, and vice versa. A threshold on a ranked list moves along a precision-recall curve as it changes.

**Precision-recall curve:** a curve plotting precision as a function of recall at each rank cutoff, summarizing a system's retrieval quality across all recall levels.

**Average precision (AP):** the mean of the precision values at each rank where a relevant document is retrieved, averaged over all relevant documents. Summarizes the precision-recall curve in a single number.

**A/B testing:** an online evaluation method that randomly splits users between two system variants (A and B) and measures which produces better user outcomes (e.g. clicks, dwell time).

**Benchmarking:** evaluating IR systems using standardized test collections (queries + relevance judgments) so that results from different systems can be compared fairly.

**Test collections:** a corpus of documents, a set of queries (topics), and relevance judgments (qrels) used for offline IR evaluation. TREC collections are the most widely used.

## Exercise 6.1

> An IR system returns 8 relevant documents, and 10 nonrelevant documents. There are a total of 20 relevant documents in the collection. What is the precision of the system on this search, and what is its recall?

The system returns:

```text
8 relevant documents
10 nonrelevant documents
18 retrieved documents total
20 relevant documents total in the collection
```

Precision:

```text
P = relevant retrieved / retrieved
  = 8 / (8 + 10)
  = 8 / 18
  = 4 / 9
  ≈ 0.444
```

Recall:

```text
R = relevant retrieved / relevant in collection
  = 8 / 20
  = 0.4
```

So precision is about `44.4%`, and recall is `40%`.

## Exercise 6.2

> Derive the equivalence between the two formulas for F measure on slide 51 (using α and β as parameters), given that α = 1/(β² + 1).

The two formulas are:

```text
F = 1 / (alpha * (1/P) + (1 - alpha) * (1/R))
```

and

```text
F_beta = ((beta^2 + 1) * P * R) / (beta^2 * P + R)
```

Given:

```text
alpha = 1 / (beta^2 + 1)
```

we also have:

```text
1 - alpha = 1 - 1 / (beta^2 + 1)
          = beta^2 / (beta^2 + 1)
```

Substitute into the first formula:

```text
F = 1 / ( 1/((beta^2 + 1)P) + beta^2/((beta^2 + 1)R) )
```

Put the denominator over a common denominator:

```text
1/((beta^2 + 1)P) + beta^2/((beta^2 + 1)R)
= (R + beta^2 P) / ((beta^2 + 1) P R)
```

Therefore:

```text
F = ((beta^2 + 1) P R) / (beta^2 P + R)
```

which is the `F_beta` formula.

## Exercise 6.3

> Must there always be a break-even point between precision and recall? Either show there must be or give a counter-example.

No, there does not have to be a break-even point where precision equals recall.

A counterexample is a ranked result list where the precision-recall points never satisfy `P = R`. Suppose there are 4 relevant documents in the collection, but the system returns only:

```text
R N N
```

The precision-recall values after each prefix are:

| rank | relevant retrieved | precision | recall |
| ---: | ---: | ---: | ---: |
| 1 | 1 | 1.000 | 0.250 |
| 2 | 1 | 0.500 | 0.250 |
| 3 | 1 | 0.333 | 0.250 |

There is no prefix with `precision = recall`. Thus a break-even point is not guaranteed.

## Exercise 6.4

> What is the relationship between the value of F1 and the break-even point?

At a break-even point, precision and recall are equal:

```text
P = R = b
```

The balanced F measure is:

```text
F1 = 2PR / (P + R)
```

Substitute `P = R = b`:

```text
F1 = 2b^2 / (b + b)
   = 2b^2 / 2b
   = b
```

So if a system has a break-even point at value `b`, then `F1` at that point is also `b`.

This does not mean the maximum `F1` over the ranked list must equal the break-even point. It only says that when `P = R`, the harmonic mean is the same value.

## Exercise 6.5

> The following list of Rs and Ns represents relevant (R) and nonrelevant (N) returned documents in a ranked list of 20 documents retrieved in response to a query from a collection of 10,000 documents. The top of the ranked list (the document the system thinks is most likely to be relevant) is on the left of the list. This list shows 6 relevant documents. Assume that there are 8 relevant documents in total in the collection.
>
> `R R N N N N N N R N R N N N R N N N N R`
>
> 1. What is the precision of the system on the top 20?
> 2. What is the F1 on the top 20?
> 3. What is the uninterpolated precision of the system at 25% recall?
> 4. What is the interpolated precision at 33% recall?
> 5. Assume that these 20 documents are the complete result set of the system. What is the MAP for the query?

The ranked list is:

```text
R R N N N N N N R N R N N N R N N N N R
```

The relevant documents occur at ranks:

```text
1, 2, 9, 11, 15, 20
```

There are 8 relevant documents total in the collection.

### Precision at top 20

There are 6 relevant documents in the top 20:

```text
P@20 = 6 / 20 = 0.3
```

So precision at top 20 is `30%`.

### F1 at top 20

Recall at top 20 is:

```text
R@20 = 6 / 8 = 0.75
```

Therefore:

```text
F1 = 2PR / (P + R)
   = 2 * 0.3 * 0.75 / (0.3 + 0.75)
   = 0.45 / 1.05
   ≈ 0.4286
```

### Uninterpolated precision at 25% recall

Since there are 8 relevant documents total, 25% recall means retrieving:

```text
0.25 * 8 = 2 relevant documents
```

The second relevant document is at rank 2:

```text
P@2 = 2 / 2 = 1.0
```

So the uninterpolated precision at 25% recall is `1.0`.

### Interpolated precision at 33% recall

The precision-recall points at relevant ranks are:

| relevant rank | rank | precision | recall |
| ---: | ---: | ---: | ---: |
| 1 | 1 | 1/1 = 1.0000 | 1/8 = 0.125 |
| 2 | 2 | 2/2 = 1.0000 | 2/8 = 0.250 |
| 3 | 9 | 3/9 = 0.3333 | 3/8 = 0.375 |
| 4 | 11 | 4/11 = 0.3636 | 4/8 = 0.500 |
| 5 | 15 | 5/15 = 0.3333 | 5/8 = 0.625 |
| 6 | 20 | 6/20 = 0.3000 | 6/8 = 0.750 |

Interpolated precision at recall `0.33` is the maximum precision at any recall level at least `0.33`.

The future points have precisions:

```text
0.3333, 0.3636, 0.3333, 0.3000
```

The maximum is:

```text
4/11 ≈ 0.3636
```

So interpolated precision at 33% recall is about `0.364`.

### MAP for the query

For a single query, MAP is the same as average precision. Average precision is the mean of precision values at the ranks where relevant documents are retrieved, divided by the total number of relevant documents.

```text
AP = (P@1 + P@2 + P@9 + P@11 + P@15 + P@20) / 8
   = (1 + 1 + 3/9 + 4/11 + 5/15 + 6/20) / 8
   ≈ 3.3303 / 8
   ≈ 0.4163
```

The denominator is 8 because there are 8 relevant documents in the collection. The two relevant documents not returned in the complete result set contribute 0.

So the MAP for this single query is approximately `0.416`.
